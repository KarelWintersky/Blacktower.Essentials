Настройка системной почты для root обычно решается двумя способами: простым (перенаправление на внешний ящик) и более сложным (полноценная отправка через внешний SMTP). Выбор зависит от того, нужно ли просто получать письма root'а или настроить отправку со своего домена.

---

### ⚙️ Способ 1: Простой алиас (перенаправление)

Этот метод самый простой и быстрый. Вся почта для пользователя `root` будет автоматически перенаправляться на указанный вами внешний email-адрес. Система не будет пытаться самостоятельно отправить письмо в интернет, а просто передаст его локальному агенту доставки (MDA) для пересылки.

Существует два основных способа это сделать:

1.  **Через файл `/etc/aliases` (рекомендуемый способ)**
    Это централизованный и стандартный способ настройки алиасов в Unix-подобных системах.

    *   **Шаг 1:** Откройте файл `/etc/aliases` с правами суперпользователя.
        ```bash
        sudo nano /etc/aliases
        # или
        sudo vi /etc/mail/aliases
        ```

    *   **Шаг 2:** Найдите в файле строку, начинающуюся с `root:` (она может быть закомментирована символом `#`). Измените или раскомментируйте её, указав ваш настоящий email-адрес.
        ```bash
        root: your.email@example.com
        ```

        *   **Важно:** Вы можете указать несколько адресов через запятую.
        *   **Пример:** `root: my@gmail.com, admin@my-company.com`
        *   **Примечание:** Также полезно настроить алиасы для стандартных почтовых ролей, таких как `postmaster` и `abuse`, направив их на root.

    *   **Шаг 3:** После сохранения файла необходимо обновить базу данных алиасов, выполнив команду `newaliases`. Без этого изменения не вступят в силу.
        ```bash
        sudo newaliases
        ```

2.  **Через файл `~root/.forward`**
    Это альтернативный метод, который действует только для пользователя `root`.

    *   **Шаг 1:** Создайте (или отредактируйте) файл с именем `.forward` в домашней директории root. Обычно это `/root/.forward`.
        ```bash
        sudo nano /root/.forward
        ```
    *   **Шаг 2:** Впишите в этот файл ваш внешний email-адрес.
        ```text
        your.email@example.com
        ```
    *   **Шаг 3:** Убедитесь, что файл принадлежит пользователю `root` и имеет правильные права доступа (обычно `644` или `600`). Некорректные права могут привести к игнорированию файла.

> **Результат:** Теперь вся почта, отправленная системой для пользователя `root`, будет приходить на ваш ящик `your.email@example.com`. Это самое простое решение для большинства администраторов.

### 🚀 Способ 2: Отправка через внешний SMTP

Этот способ сложнее, но и значительно мощнее. Вы настраиваете свою систему как полноценного почтового клиента, который подключается к внешнему SMTP-серверу (например, Gmail, Яндекс.Почта или SMTP-релей вашего хостинга) и отправляет письма через него. Это полезно, если вам нужно, чтобы письма отправлялись **от вашего домена** (`cron@my-domain.com`), а не от какого-то локального пользователя. Стандартные MTA (Sendmail, Postfix) могут быть избыточны для этой задачи, поэтому часто используют легковесные утилиты, такие как `msmtp` или `ssmtp`.

#### Инструкция по настройке `msmtp` (пример для Ubuntu/Debian)

`msmtp` — это простой и современный клиент SMTP, который идеально подходит для этой роли.

*   **Шаг 1: Установка `msmtp` и вспомогательных утилит**
    ```bash
    sudo apt update
    sudo apt install msmtp msmtp-mta mailutils
    ```
    Пакет `msmtp-mta` эмулирует стандартную команду `sendmail`, чтобы все системные скрипты и службы (например, `cron`) могли отправлять почту через `msmtp`. Пакет `mailutils` содержит программу `mail` для тестирования.

*   **Шаг 2: Создание конфигурационного файла `msmtp`**
    Создайте файл `/etc/msmtprc` и настройте его для вашего SMTP-сервера.
    Пример конфигурации для Gmail (`smtp.gmail.com:587`):

    ```bash
    sudo nano /etc/msmtprc
    ```

    Добавьте следующие строки:
    ```text
    # Настройки по умолчанию для всех аккаунтов
    defaults
    auth           on          # Включить аутентификацию
    tls            on          # Включить TLS
    tls_trust_file /etc/ssl/certs/ca-certificates.crt
    logfile        /var/log/msmtp.log

    # Описание вашего аккаунта на внешнем SMTP-сервере
    account        gmail
    host           smtp.gmail.com
    port           587
    from           your-email@gmail.com
    user           your-email@gmail.com
    password       your-google-app-password  # Используйте пароль приложения!

    # Установить этот аккаунт как используемый по умолчанию
    account default : gmail
    ```

    **Крайне важно! Для Gmail и многих других сервисов нужно использовать "пароль приложения", а не ваш обычный пароль. Для Gmail это настраивается в настройках безопасности аккаунта.**

*   **Шаг 3: Защита конфигурационного файла**
    Так как в файле хранится пароль, необходимо ограничить к нему доступ.
    ```bash
    sudo chmod 600 /etc/msmtprc
    ```

*   **Шаг 4: Настройка алиасов для `msmtp`**
    `msmtp` также использует алиасы для преобразования локальных пользователей в email-адреса. Создайте файл `/etc/aliases` в том же формате, что и в Способе 1:
    ```bash
    sudo nano /etc/aliases
    # Пример: перенаправляем всю почту для root на внешний ящик
    root: your.email@gmail.com
    default: your.email@gmail.com
    ```
    Затем укажите путь к этому файлу в `/etc/msmtprc`, добавив строку в секцию `defaults` или `account`:
    ```text
    aliases /etc/aliases
    ```

*   **Шаг 5: Тестирование отправки**
    Проверьте, работает ли настройка:
    ```bash
    echo "Тело письма" | mail -s "Тема письма" root
    ```
    Проверьте лог `msmtp` на наличие ошибок:
    ```bash
    sudo cat /var/log/msmtp.log
    ```
    Посмотрите на почтовый ящик, указанный в алиасе `root`. Письмо должно прийти, а в заголовке `From` будет указан адрес из параметра `from` в конфигурации `msmtp` (в примере — `your-email@gmail.com`). Если нужно изменить `From` на что-то вроде `root@my-domain.com`,`rewriteDomain` в конфигурации.

### 📋 Итоговое сравнение

| Характеристика | Алиас (`.forward`/`/etc/aliases`) | Внешний SMTP (`msmtp`) |
| :--- | :--- | :--- |
| **Сложность** | Очень низкая | Средняя |
| **Зависимости** | Только локальный MTA (обычно уже есть) | Нужна установка `msmtp` или аналога |
| **Кому доставляется** | На указанный внешний ящик (как получатель) | Отправляется **от имени** настроенного пользователя |
| **Обратный адрес (From)** | `root@localhost` или `root@hostname` | Настроенный email (например, `user@gmail.com`) |
| **Использование** | Просто получать уведомления на личную почту | Нужна отправка с реального домена, интеграция с CRM и т.д. |

Выберите способ, который лучше соответствует вашим задачам. Для 90% задач администратора будет достаточно первого, более простого способа с алиасом.
